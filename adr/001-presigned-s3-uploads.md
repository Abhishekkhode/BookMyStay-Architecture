# ADR-001: Direct-to-S3 Uploads via Pre-Signed URLs

## Status
Accepted (v1)

## Context
Manager-uploaded images (rooms, listings) need to be stored in S3. The naive approach routes the file through the backend: browser → EC2 → S3.

## Decision
The browser uploads directly to S3 using short-lived (2-minute) pre-signed URLs generated by the backend.

## Why
Routing uploads through EC2 spends compute and bandwidth moving bytes the server doesn't need to touch, and turns the application server into a bottleneck under load. Pre-signed URLs let S3 handle the transfer directly while the backend retains control at the point of URL generation — not at the point of transfer.

Controls enforced at generation time:
- 50MB file size limit
- Redis-backed rate limit: 5 uploads/day per manager account
- S3 CORS policy restricted to known origins (Vercel + local dev)

## Consequences
- EC2 stays lightweight; upload traffic doesn't touch the app server at all.
- Rate limiting and size limits exist only at issuance — once a URL is issued, S3 itself doesn't re-check those rules. Acceptable for v1 given short URL expiry and per-manager limits; would need closer review under adversarial conditions.
